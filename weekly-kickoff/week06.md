## 프로파일러 — 무엇을 보는가

```text
🔥 CPU       — 어떤 메서드가 가장 오래 도나
💧 메모리    — GC 빈도 / heap 사용량
🚪 IO        — 디스크/네트워크 대기
🎯 락 경합   — 동시성 대기 시간
```

```text
flamegraph = 함수 호출 깊이를 _가로 막대_ 로
가장 _넓은_ 막대 = 가장 오래 걸린 함수
```

# 우선순위 — 임팩트 × 노력

```text
        임팩트 ↑
            │
   ┌────────┼────────┐
   │  먼저! │ 좋지만 │
   │  (큰 임팩트, │  (큰 임팩트,│
   │  적은 노력) │  큰 노력) │
   ├────────┼────────┤
   │  스킵  │  나중  │
   │  (작은) │  (큰 노력)│
   └────────┼────────┘
            │ 노력 →
```

> 핫스팟 3 개 중 _먼저!_ 칸의 1 개를 evidence 에.

# 함정 — 흔한 실수

- ❌ "느려요" 만 → ✅ "p99 1.2s, 평균 600ms" 수치 박기

- ❌ 프로덕션 _live_ 에 프로파일러 강하게 켜기 → ✅ 샘플링 모드
