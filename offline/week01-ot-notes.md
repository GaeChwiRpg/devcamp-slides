# Week 1 OT — 발표 노트

> 슬라이드별 멘트 가이드. 시간 박스 = 60–90분. Discord 라이브 동시 진행.

## 1. 표지 (1분)

- "오늘은 10주 전체 그림. 다음 주 토 14:00 부터 매주 한 챕터씩 깊이 파고들어요."
- 늦은 분 기다리지 않고 _바로_ 시작 (시간 약속의 신호)

## 2. 오늘의 흐름 (1분)

- 5블록 짧게 안내. 질문은 끝에 모아서.

## 3. 왜 이 부트캠프인가 (3분)

- _한 문장 메시지_ 강조: "본인 입으로 본인 코드를 설명할 수 있다"
- 시중 강의들과의 차이 = _말로 답할 수 있는 결과물_

## 4. 우리가 깨야 할 4가지 (4분)

- 4가지 항목 각각 학생 한 명 호명해서 _공감_ 끌어내기 (선택)
- 너무 부정적으로 가지 말고 "이게 일반적, 우리는 이걸 깬다"

## 5. 해결 — 3축 압축 (3분)

- 표 짚어가며 "다음 슬라이드부터 각 축을 펼친다"

## 6-7. 10주 RPG Map (3분)

- 미션 ID 는 _이미 정해진_ 것. 주차마다 어떤 _아이템_ 을 얻는지 RPG 메타포로 강조
- W10 = FINAL BOSS = 면접

## 8. 6 공통 필수 기능 (4분)

- 각 항목 한 줄씩만 — _깊이_ 는 매주 슬롯에서
- "팀 프로젝트는 이 6개를 _최소 1번씩_ 다 커버하도록 설계"

## 9. 라이프사이클 5 단계 (4분)

- 산출물 표 강조 — _남기는 것_ 이 면접 답변의 근거
- "기획 = PRD, 운영 = MONITORING.md ... 단어가 머릿속에 남으면 충분"

## 10. AI Native 4 요소 (3분)

- _선택_ 이 아니라 _필수_. AI 도구를 _안 쓰는_ 학생은 이 부트캠프에서 손해
- 다만 검증 루프 = AI 답을 _믿지 않고_ 본인이 검증

## 11. 평가 5축 (4분)

- 표를 그대로 짚어가며 1점 / 3점 / 5점 차이 명확히
- 0.5 단위라는 점 강조 — 작은 진전도 점수에 반영

## 12. 분기점 5개 (3분)

- W4 / W8 이 _가장 중요한 두 분기점_
- _매주_ 통과 조건은 ≥3.0 = 통과 가능 수준. 너무 부담 갖지 않게.

## 13. 팀 매칭 룰 (3분)

- 이미 팀 매칭은 사전 완료 상태로 가정 — 본인 팀 발표
- 분담 sample 링크 — _참고만_, 강제 아님

## 14. 제출 시스템 개요 (3분)

- 각 학생이 본인 GitHub 레포를 _혼자_ 가진다는 점 명확화
- mission-guard / AI 리뷰는 _자동_ — 사람 기다리지 않음

## 15. Discord 코호트 채널 안내 (2분)

- 채널 매핑 표 짚으면서 본인 코호트 카테고리 직접 보여주기
- 오피스아워 시간 (화·목 21:00) 한 번 더 강조

## 16. 제출 흐름 6단계 (3분)

- _라이브 시연 안 함_ — 6단계 텍스트로 짚어가며 설명
- 학생들이 _첫 PR_ 만들 때 흐름이 그림으로 머리에 남도록

## 17. mission-guard 가 막는 것 (2분)

- 4가지 차단 케이스 학생들이 _자주_ 빠지는 함정
- 형식 검증 통과 == AI 리뷰 시작 ≠ 점수 보장

## 17. 운영 리듬 (3분)

- 토요일이 _수업_ 이 아니라 _이벤트_ 라는 점 강조
- 평일 / 주말의 책임 분담 — 학생은 평일 손, 토요일 머리

## 18. 도구 스택 (2분)

- _권장_ 순서대로 — Claude Code 가 가장 큰 무기
- Discord 채널 운영 시간대만 명확히

## 19. 학생 측 도구 (3분)

- _포트폴리오 1조각_ 메타포 강조
- 매주 1조각 ≠ 매주 미션 통과. _매주 1조각이 본인 머리에 남으면 OK_

## 20. 첫 주 미션 (2분)

- 미션 ID, 마감일, _다음 주 토 14:00_ 에서 깊게 다룬다는 점만
- 오늘 OT 안에서는 _요약_ 만

## 21. 첫 주 할 일 (3분)

- 4개 체크박스 — _이번 주 안에_ 완료해야 다음 주 토에 따라온다
- 환경 셀프체크 3종은 _가장_ 중요. 이게 막히면 코드 못 짠다.

## 22. 약속 (3분)

- 운영진 / 학생 양쪽 약속 _말로_ 한 번씩 짚어준다
- 학생 측 "AI 답을 믿지 않고 검증" 부분이 핵심 — 강조

## 23. 마무리 — Press Start (1분)

- "1주 / 5주 / 10주 후의 모습" 3 단계로 시각화
- 다음 주 토 14:00 약속 한 번 더

## 네트워킹 (15–30분)

- 슬라이드 끝나면 자유 네트워킹
- 운영진은 팀별로 한 명씩 인사 + 본인 도메인/배경 듣기
- 사진 1장 (전체 + 팀별)
