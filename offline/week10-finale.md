# 면접 답변 공식 ① — P-O-D-A-R

5장 답변 흐름.

| 단계 | 내용 |
| --- | --- |
| **P**roblem | 무엇이 문제였나 (수치 1개) |
| **O**ptions | 고려한 옵션 2~3개 |
| **D**ecision | 어느 것을 선택, 왜 |
| **A**ction | 본인이 _직접_ 한 것 |
| **R**esult | 결과 수치 (before/after) |

매주 evidence 의 _숫자_ 가 P 와 R 의 근거.

> Week 10 책 베이스. 이력서 bullet 1개당 P-O-D-A-R 한 세트.

---

<!-- _class: lesson -->

# 면접 답변 공식 ② — A-B-C

3 단계 압축.

| 단계 | 내용 |
| --- | --- |
| **A** | 결론 (한 줄) |
| **B** | 근거 + 비교 (다른 옵션 대비 _왜 이게_) |
| **C** | 본인 경험 (이 부트캠프에서 _직접 했던_ 사례) |

> 면접 30초~1분 답변에 적합. P-O-D-A-R 의 압축 버전.

---

# 답변 시뮬레이션

**Q**: "동시성 이슈를 어떻게 해결하나요?"

**A** (결론): 비관적 락 또는 낙관적 락 중 도메인 특성에 맞춰.

**B** (비교): 재시도 비용 vs 락 경합 비용 — 재고 차감처럼 _경합_ 이 잦으면 비관적, 좋아요처럼 _드물면_ 낙관적.

**C** (경험): Week 5 미션에서 `@Lock(PESSIMISTIC_WRITE)` 로 동시 50요청 처리. 음수 재고 0건. evidence/concurrency-success-log.md 에 기록.
